## Architecture Decisions
- Decision: keep `/reports` as redirect compatibility route
  - Why: preserve existing links while moving product UX to unified suggestions flow
  - Tradeoff: temporary duplicate route surface
- Decision: apply red-flag triage before meal-dependent recommendation generation
  - Why: emergency escalation must not depend on meal history
  - Tradeoff: lightweight rule gate in router path until full application-layer extraction
- Decision: add `scope=self|household` to suggestions list/detail
  - Why: reuses household model for shared visibility without breaking default self behavior
  - Tradeoff: router currently merges member records; dedicated read-model can optimize later
