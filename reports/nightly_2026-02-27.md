## Architecture Decisions
- Decision: enforce red-flag triage at API service boundary before meal-dependent recommendation generation
  - Why: emergency symptom escalation must not depend on meal history availability
  - Tradeoff: adds a lightweight rule-based gate in router flow pending deeper application-layer extraction
- Decision: return `200` with structured `safety.decision = "escalate"` for red-flag text
  - Why: keeps deterministic response contract while clearly signaling urgent-care path
  - Tradeoff: client must inspect payload state (not just HTTP status) for escalation UX
- Decision: include `safety` object in persisted suggestion payload
  - Why: improves auditability and future household-sharing compatibility
  - Tradeoff: additive schema expansion for suggestions records
